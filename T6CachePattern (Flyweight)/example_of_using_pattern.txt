Контекст проблемы:
Система рендеринга деревьев в стратегической игре, где тысячи деревьев имеют одинаковые типы, текстуры и визуальные характеристики.
Создание отдельного объекта для каждого дерева с полным набором атрибутов приводит к чрезмерному потреблению памяти из-за дублирования одинаковых данных.

Реализация:
1.	Разделяемое состояние (внутреннее):
•	Тип дерева ("Дуб", "Сосна", "Берёза")
•	Текстура и 3D-модель
•	Основной цвет и материалы
•	Звуки взаимодействия
•	Физические свойства
2.	Внешнее состояние (уникальное):
•	Координаты на карте (x, y, z)
•	Состояние (целое/поврежденное)

Пример использования:
На игровой карте 5000 деревьев 3 видов:
• 	2000 дубов
• 	2000 сосен
• 	1000 берёз
Все деревья одного типа имеют:
•	Одинаковую текстуру: "дуб_текстура.png"/"сосна_текстура.png"/"берёза_текстура.png"
•	Модель: "дуб_меш.obj"/"сосна_меш.obj"/"берёза_меш.obj"
•	Звук рубки: "дуб_рубка.wav"/"сосна_рубка.wav"/"берёза_рубка.wav"
•	Цвет: "темно-зеленый с коричневым стволом"/"хвойно-зелёный с серовато-коричневым стволом"/"светло-зеленый с белым стволом"
Без паттерна создается 5000 объектов с дублирующими графическими ресурсами. С паттерном создается 3 объекта (тип "Дуб", тип "Сосна", тип "Берёза"), которые переиспользуются для всех деревьев этого типа.
